Clean Comparison (Class & Exam Reference)

Model-based evaluation: what each model explains vs. predicts
Model Category What it Explains What it Predicts
Norman Cognitive / descriptive Understanding, errors, feedback, gulfs ❌ Nothing
GOMS Predictive / analytic Task structure for expert users ✔ Relative efficiency
KLM Predictive / quantitative Low-level physical and motor actions ✔ Execution time

Worked Example: ATM Cash Withdrawal

Task: Withdraw cash from an ATM machine using a card and PIN.

ATM Cash Withdrawal: Model Comparison
Model Focus Example Actions Explains / Predicts
Norman Cognitive / descriptive See ATM screen → Decide amount → Insert card → Enter PIN → Confirm → Take cash Explains errors, feedback, mental steps, gulfs
GOMS Predictive / analytic Goal: Withdraw cash → Operators: insert card, type PIN, press buttons → Method: follow sequence → Selection: correct amount Predicts efficiency, correct sequence for expert users
KLM Execution / quantitative H → P → M → K → B → R → P B (take cash) Predicts exact time for each action

KLM operator sequence for ATM withdrawal:

H[hand → card]
P[insert card]
M[decide amount]
K[PIN digits]
B[press confirm]
R[ATM processes]
P B[take cash]

Key Takeaways

Norman explains cognition and errors, GOMS predicts task sequence for efficiency, and KLM models low-level execution time.
